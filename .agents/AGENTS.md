# Правила и Инструкции для ИИ-ассистентов по проекту StreamLume

Перед началом работы вы ОБЯЗАНЫ соблюдать эти правила (от 25.06.2026).

### 🛠️ ПРАВИЛА СБОРКИ И ЗАВИСИМОСТЕЙ
1. **Зависимости Expo:** ЗАПРЕЩЕНО использовать `npm install` для установки нативных библиотек. ВСЕГДА используйте `npx expo install [package]`. При поломках зависимостей (особенно конфликтах версий SDK при сборке) начинайте диагностику с команды `npx expo install --fix`.
2. **Блокировки интерфейса:** ЗАПРЕЩЕНО делать "вечные загрузки". Любое ожидание данных (от API, AsyncStorage, Zustand) ОБЯЗАТЕЛЬНО должно иметь fallback-таймаут (1.5-2 секунды). По истечении этого времени интерфейс должен разблокироваться, чтобы пользователь никогда не видел зависший спиннер из-за переменной `hasHydrated`.
3. **Тестирование релизов:** На Android-системах версия для публикации (APK/AAB) собирается локально через Gradle. Всегда проверяйте работоспособность (особенно гидратацию состояний) на собранной локально Release-версии, так как среда разработки (`npm start`) может маскировать многие ошибки, возникающие при минификации кода.
4. **Безопасность разработки:** При внедрении крупных изменений для других платформ (например, адаптация под TV) изолируйте их и проверяйте, чтобы они ни в коем случае не ломали основную рабочую логику мобильного приложения.
